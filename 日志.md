# 开发日志
## 开发目标
输入同步 + 预测 + 回滚（Rollback Netcode），典型由 GGPO 推广
关键点是“本地输入立即生效”，远端输入来晚了就预测，预测错了就回滚重演
时间线： 
仿真帧 frame_id（离散）：固定 60Hz，所有逻辑只按帧推进。
输入缓冲 InputBuffer：每个玩家每帧的输入（本地真实、远端真实、远端预测）。
状态缓冲 StateBuffer：保存最近 N 帧的 WorldState，用于回滚恢复与重演。

开发路线
Lab 0：项目骨架 + 固定帧循环
Lab 1：离线“确定性”最小格斗仿真（必须）
Lab 2：WorldState 可保存/恢复 + hash 校验

Lab 3：UDP 本机互联 + 网络统计（先通）
Server 权威推进（客户端先不回滚，先验证链路）：
Client：每 tick 发输入（带冗余）
Server：按 tick 推进世界（缺输入时用默认输入/或等待策略）

Server：回传 ACK（“我处理到哪个 tick 了”）或低频回传 state hash
Lab 4：输入同步“窗口冗余”（格斗输入可靠性的常用办法）
Lab 5：预测（Prediction）+ 回滚重演（Rollback）
